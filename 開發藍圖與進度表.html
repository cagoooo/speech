v2.8 推薦開發優先順序分析

雲端資料已就緒，現在最有價值的是讓學生「看見自己的進步」。以下按影響力排序的三週衝刺計畫：

🔥 第一週（高影響力 × 低成本）
  • ⭐ 個人進步折線圖：Chart.js + 已有雲端資料，最能讓學生「看到自己在變好」的關鍵功能
  • ⭐ 語速自動計算：邏輯簡單（字數/秒數*60），但對學生的指導價值極大
  • ⭐ 題目難易度標籤：低成本高回饋，可用 AI 批次自動分類
⚡ 第二週（AI 深化）
  • 口頭禪偵測：演說訓練最容易被忽略的失分點，學生自己不會察覺
  • 演說結構比例視覺化：開場／主體／結語配重是常見失分項，圖示化讓學生一眼看懂
  • 全文朗讀功能：Web Speech API 幾行程式即可完成，學生實用性高
📋 第三週（黏著度）
  • 成就徽章系統：心理學研究證實可提升 40%+ 的回訪率
  • 練習連勝日曆：GitHub 綠方格風格，讓學生一年累積成就一目了然
  • 深色模式 + 視窗鎖定：兩項 UX 打磨，合計不到 1 天

  • 觸控手勢支援：手機上左右滑動切換分頁、下拉刷新歷史紀錄
🔒 資料安全與隱私保護
  • API 金鑰加密儲存：localStorage 中的 Gemini Key 以 AES 加密，防止開發者工具直接讀取
  • Supabase RLS 已實作：所有查詢自動過濾，確保只能看到自己的資料（已完成✓）
  • 音訊資料即時清除：錄音分析完成後立即從記憶體釋放
  • Content Security Policy（CSP）：限制只允許 googleapis.com / supabase.co 外部請求
  • 匯出 CSV/JSON 時加入警示：提醒老師妥善保管，不得上傳至公開雲端
  • 學生資料刪除權：提供「永久刪除我的所有資料」按鈕，符合 GDPR / 個資法精神
  • 定期滲透測試：每季進行一次 OWASP Top 10 檢測
🧪 品質保證與測試策略
  • 跨瀏覽器測試：Chrome（主）、Edge（支援）、Firefox（部分）、Safari（iOS 常用）
  • 跨裝置測試：Windows 筆電（班級電腦）、iPad（老師平板）、iPhone（學生手機）
  • AI 回應品質監控：記錄 Gemini 回應時間分布，設 10 秒超時自動重試
  • 使用者測試計劃：3-5 位老師 Think-aloud 測試，找出介面操作盲點
  • Lighthouse 分數目標：Performance ≥ 90、Accessibility ≥ 90、Best Practices ≥ 95
  • E2E 自動化測試：Playwright 腳本跑一遍主流程，確保重構不壞功能
  • A/B 測試框架：不同的 UI 文案、按鈕位置實驗哪個轉換率高
🚨 技術風險與已知限制

⚠️ Supabase 免費方案限制

500MB 資料庫、2GB 頻寬/月、50K MAU。若使用人數超過，需升級 Pro ($25/月)。建議在 400 人同時使用前規劃升級，或改用多個專案分流。

💡 解方：加入資料保留政策，老資料歸檔到外部儲存

⚠️ Gemini API 配額限制

免費方案每分鐘 15 次請求、每天 1,500 次；全班 40 人同時用 AI 評析可能會超過。生產環境應申請 Paid tier 或加入排隊機制。

💡 解方：前端加入 rate limit 提示，失敗時自動排隊重試

⚠️ Web Speech API 瀏覽器相容性

只有 Chrome/Edge 完全支援；Safari iOS 有限，Firefox 不支援。學生用不同裝置可能體驗不一致。

💡 解方：整合 Google Cloud Speech-to-Text 作為備援（付費）

⚠️ Google OAuth redirect URL 鎖定

目前硬編碼 https://cagoooo.github.io/speech/，若未來搬遷網域需同時更新 Google Cloud Console、Supabase Auth、前端三處設定。

💡 解方：以環境變數管理 redirect URL，部署時動態注入

⚠️ 單一 HTML 檔體積將過大

目前 index.html 已超過 260KB（約 4,300 行），再擴充功能容易超過瀏覽器優化閾值，首次載入會變慢。

💡 解方：v3.0 改用 Vite + Vue 元件化，按需載入

⚠️ 練習紀錄無 offline 支援

目前學生沒網路時無法記錄練習；練習內容完全遺失。校園網路不穩時影響體驗。

💡 解方：v2.9 加入 IndexedDB 離線佇列 + Service Worker
