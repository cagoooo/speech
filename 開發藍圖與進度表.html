推薦開發優先順序分析

基於「對老師和學生的即時幫助」與「開發成本」兩大維度，以下是建議的實作順序。標示 ⭐ 的為最推薦優先實作的項目。

🔥 第一優先（本週）
  • ⭐ AI 串流輸出：逐字顯示生成結果，消除等待焦慮，體驗感提升最大
  • ⭐ 演說評分量表：老師最需要的打分工具，直接對應比賽評分表
  • ⭐ 逐字稿一鍵複製：1 小時完成，老師每天都會用到
⚡ 第二優先（下週）
  • 鍵盤快速鍵：Space 控制計時，大幅提升操作效率
  • 深色模式：CSS 變數已就緒，半天完成，護眼使用
  • CSV 題庫匯出入：老師可在 Excel 整理後批次匯入
📋 第三優先（月內）
  • 練習歷史紀錄：讓學生看見成長，是長期黏著的關鍵
  • 個人進步折線圖：搭配紀錄系統，視覺化成長軌跡
  • 口頭禪偵測：演說訓練的核心痛點之一

🔬 深度技術優化與 AI 進化建議
🧠 AI Prompt 工程深化
  • 為不同年齡層（低/中/高年級）設計差異化 Prompt，低年級語氣溫柔簡單，高年級要求更精準深入
  • 加入「Few-shot 示例」提供 2–3 個優質演說大綱範例，引導 Gemini 輸出格式更一致
  • 評析 Prompt 加入「正向優先」指令，確保先給 2 個真誠優點再談改進，符合教育心理學原則
  • 「名言推薦」Prompt 加入年齡適當性過濾，確保金句是國小學生能理解且有共鳴的語句
  • 建立 Prompt 版本管理機制，各功能 Prompt 統一存於設定檔，方便老師自訂調整措辭風格
  • 加入 System Prompt 角色設定：「你是一位資深國語演說教練，擅長以鼓勵方式引導國小學生進步…」
🤖 語音分析技術升級
  • 整合 Web Audio API AnalyserNode，錄音時即時分析音量 RMS 值，評估聲音投射是否足夠
  • 偵測靜音段（Silence Detection）：分析停頓次數與平均停頓時長，過多長停頓給予提示
  • 音高（Pitch）估算：Autocorrelation 演算法估算基頻，判斷語調是否單調，鼓勵抑揚頓挫
  • Gemini 音訊直接評析升級：發送錄音片段給 Gemini，讓 AI 同時分析聲學特徵與文字內容
  • 語音辨識信心值顯示：低信心值的字詞以橙色標記，提示學生發音需加強
⚡ 效能與載入速度優化
  • Lazy Load 分頁內容：切換頁籤時才載入對應 JS 模組，首屏時間縮短 40% 以上
  • Chart.js 改為按需引入（Tree Shaking），只引入需要的圖表模組，減少體積約 60%
  • AI 回應串流輸出（Gemini Streaming API）：逐字顯示生成結果，告別等待空白頁面
  • 音頻處理使用 Web Worker 在背景執行，避免阻塞主執行緒導致 UI 凍結
  • 加入 Loading Skeleton 骨架屏，AI 載入中時顯示佔位動畫，有效降低等待焦慮感
🎨 UX 使用體驗深化
  • 全鍵盤無障礙操作（WCAG 2.1 AA 標準）：所有功能可用 Tab/Enter/Space 操作
  • 動畫偏好尊重：偵測 prefers-reduced-motion，勝利彩帶等動畫可動態關閉
  • 錯誤狀態設計強化：API 失敗時顯示具體原因（金鑰過期/配額用盡/網路斷線）而非通用錯誤訊息
  • 引導式新手教學（Onboarding Tour）：首次使用以步驟提示框帶領老師完成完整流程
  • 破壞性操作「2 步驟確認」：刪除題目、清空紀錄先點擊→再倒數 3 秒確認，防止誤觸
  • 印刷版面精修：列印時加入頁首（學校名稱、日期）與頁尾（頁碼），每頁斷點優化
🔒 資料安全與隱私保護
  • API 金鑰加密儲存：localStorage 中的 Gemini Key 以 AES 加密，防止開發者工具直接讀取
  • 學生資料本地優先：所有練習紀錄僅存本機，不傳送至任何第三方伺服器（符合 PDPA 精神）
  • 音訊資料即時清除：錄音分析完成後立即從記憶體釋放，不緩存任何音訊數據
  • Content Security Policy（CSP）設定：限制只允許 googleapis.com 的外部請求，防止 XSS 注入
  • 匯出 CSV/JSON 時加入學生資料警示，提醒老師妥善保管，不得上傳至公開雲端
🧪 品質保證與測試策略
  • 跨瀏覽器測試清單：Chrome（主）、Edge（支援）、Firefox（部分功能）、Safari（iOS 常用）
  • 跨裝置測試：Windows 筆電（班級電腦）、iPad（老師平板）、iPhone（學生手機）
  • AI 回應品質監控：記錄 Gemini 回應時間分布，設定 10 秒超時自動重試機制
  • 使用者測試計劃：安排 3–5 位老師進行 Think-aloud 測試，找出介面操作盲點
  • Lighthouse 分數目標：Performance ≥ 90、Accessibility ≥ 90、Best Practices ≥ 95

📋 課堂練習流程（建議 40 分鐘）
  • 暖身 5 分鐘：老師用「主題建議」功能讓全班一起討論今日題目方向，激發思考
  • 準備 10 分鐘：學生個別用平台抽題 → 啟動準備計時 → 在紙上列大綱（模擬比賽）
  • 上台 5 分鐘：一位學生上台演說，老師用「演說計時」即時計時 ＋ 同步錄音辨識
  • 回饋 10 分鐘：全班用 AI 評析結果一起討論，老師補充指導重點
  • 練習 10 分鐘：其他同學兩兩一組，用 AI 大綱輔助各自練習，互相給回饋
🏠 家長配合練習建議
  • 每日 15 分鐘：用平台隨機抽題 → 準備 5 分鐘 → 演說一次 → 錄音自評

  • 週末加強：用「名言金句」功能豐富演說素材，週六練舊題加深、週日練新題拓展
  • 模擬考核：賽前兩週每天用「完整逐字稿」功能打磨定稿，計時精確控制在 4 分 30 秒
  • 家長信溝通：老師每週用「家長通知信」功能一鍵生成練習摘要，家長掌握進度
